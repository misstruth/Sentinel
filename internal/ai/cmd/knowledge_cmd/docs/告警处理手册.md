# 服务下线
告警解释：服务下线可能因为服务panic，导致pod重启造成的
解决方案：
1. 根据关键字"panic"进行最近1小时的日志搜索
2. 根据panic日志内容分析是什么bug导致的panic

# 接口失败率过高
告警解释：接口失败率过高可能是由于服务调用异常或者下游服务不可用导致的
解决方案：
1. 根据接口名和"response"关键词进行最近1小时的日志搜索
2. 根据查询到的日志的error进行分析是什么原因导致的接口失败

# 与下游对账发现差异
告警解释：与下游对账发现差异可能是由于数据同步异常或者计算错误导致的
解决方案：
1. 根据关键字"error"和"reconciliation"进行最近1小时的日志搜索
2. 根据查询到的日志内容分析是什么原因导致的对账差异

# 发现服务地域与资源地域不匹配
问题解释：在计费数据处理中，我们发现部分业务由于使用了错误的MQ队列把资源事件错投递到其他地域，造成资源所在地域与计费的服务地域不匹配。
解决方案：
1. 根据关键字"region mismatch"进行最近1小时的日志搜索
2. 根据查询到的日志内容，汇总出调用方和错误地域名

# 服务错误码与常见原因
- 12000000001：调用接口参数错误（如:类型不匹配）
- 12000000002：数据库更新失败（数据库的问题，建议排查日志）
- 12000000003：下游接口错误（下游接口报错）
- 12000000004：实例不存在（上游传了一个错误的实例id）
